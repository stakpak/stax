---
title: stax inspect
description: Display artifact metadata, identity, and layer information
---

Display an artifact's metadata, identity, persona, runtime configuration, and layer breakdown. Use this to understand what's inside an artifact before pushing, pulling, or materializing it.

## Usage

```bash
stax inspect ghcr.io/myorg/agents/backend-engineer:3.1.0
stax inspect ghcr.io/myorg/agents/backend-engineer:3.1.0 --json
stax inspect sha256:a3f8c1…e7d2
```

Accepts registry references (with tag or digest) or local cache digests.

## Options

| Flag     | Description                             |
| -------- | --------------------------------------- |
| `--json` | Output full metadata as structured JSON |

## Example output

### Human-readable

```text
Agent: backend-engineer v3.1.0
Runtime: claude-code (claude-opus-4-1)
Persona: maya-chen (Maya Chen, Senior Backend Engineer)

Layers:
  config     application/vnd.stax.config.v1+json          1.5 KB
  prompt     application/vnd.stax.prompt.v1+markdown      2.1 KB
  persona    application/vnd.stax.persona.v1+json         0.8 KB
  mcp        application/vnd.stax.mcp.v1+json             0.5 KB
  skills     application/vnd.stax.skills.v1.tar+gzip      4.3 KB  (3 skills)
  rules      application/vnd.stax.rules.v1.tar+gzip       1.7 KB  (4 rules)
  secrets    application/vnd.stax.secrets.v1+json         0.3 KB  (3 secrets)

Packages:
  @acme/compliance-rules  sha256:f8a1c3…
  @acme/mcp-bundle        sha256:4d2e7b…

Total: 11.2 KB (7 layers)
```

### JSON output

The `--json` flag returns the full OCI manifest, config blob, annotations, and layer metadata as structured JSON:

```bash
stax inspect ghcr.io/myorg/agents/backend-engineer:3.1.0 --json | jq '.config.persona'
```

This is useful for scripting, CI pipelines, and building tools on top of stax artifacts.

## What you can learn from inspect

- **Identity** — Agent name, version, and persona
- **Runtime** — Target runtime and model
- **Layer breakdown** — Each layer's media type, size, and content summary
- **Dependencies** — Resolved packages with their locked digests
- **Annotations** — OCI manifest annotations (build timestamp, source URL, etc.)
